# Implementation Workflow Requirements

Each repo-local `IMPLEMENTATION_WORKFLOW.md` should tell the implementation agent:

1. how to read the issue from Linear and confirm it is still in `Start`
2. how to claim the issue by moving it to `Implementing`
3. how to prepare or validate the issue branch and worktree
4. what coding and verification standards apply in that repo
5. which tests, checks, or build commands are mandatory before handoff
6. when to move the issue to `Review`
7. when it is acceptable to move directly to `Deploy`
8. when to move the issue to `Human Needed`
9. what short Linear status updates or comments are required during work
