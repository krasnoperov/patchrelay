# Development Workflow Requirements

Each repo-local `DEVELOPMENT_WORKFLOW.md` should tell the development agent:

1. how to confirm the issue is still in the development-ready Linear state
2. how to work only inside the provided issue worktree
3. what coding, testing, and validation standards apply in that repository
4. what evidence must be left in the thread for a read-only report
5. when the issue should advance to `Review`
6. when it may skip directly to `Deploy`
7. when it must move to `Human Needed`
8. what concise handoff summary must be included for the next stage
