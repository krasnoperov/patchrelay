# Review Workflow Requirements

Each repo-local `REVIEW_WORKFLOW.md` should tell the review agent:

1. how to confirm the issue is ready for review
2. how to inspect the existing workspace, branch, and prior stage evidence
3. what correctness, regression, and release-risk checks are required
4. what commands or validations must be rerun during review
5. what findings and approvals must be captured in the thread
6. when review should return the issue to development
7. when review may advance the issue to deploy
8. when review must pause and move the issue to `Human Needed`
