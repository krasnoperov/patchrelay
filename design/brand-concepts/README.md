# PatchRelay Brand Concepts

Developer-first image explorations for PatchRelay's repo surfaces.

These concepts are intentionally repo-native and editable:

- `service-logo-modern-orbit.svg` - modern infrastructure mark with routing/orchestration cues
- `service-logo-cli-badge.svg` - terminal-forward badge for CLI-leaning contexts
- `github-project-social-blueprint.svg` - 1280x640 social preview built from system flow diagrams
- `promo-hero-signal-grid.svg` - landing graphic with queues, branches, and reactive loop motion
- `promo-hero-checklist-ops.svg` - landing graphic centered on operator confidence and checklist flow
- `raster/marketing-hero-control-room.png` - rendered marketing hero with control-room and signal-routing energy
- `raster/marketing-hero-operator-desk.png` - rendered marketing hero with checklist and operator-desk energy
- `index.html` - quick gallery to compare the directions side by side

Suggested fit:

- primary service logo: `service-logo-modern-orbit.svg`
- alternate CLI/community mark: `service-logo-cli-badge.svg`
- GitHub social/project preview: `github-project-social-blueprint.svg`
- promo site hero directions: compare the two `promo-hero-*.svg` variants
- raster marketing options: compare the two `raster/marketing-hero-*.png` variants

Raster notes:

- full-size renders are `1536x1024`
- web-sized copies are `1440x960` with the `-web.png` suffix
- the SVG files remain the editable source-history directions; the raster files are presentation-ready marketing concepts

The concepts are biased toward PatchRelay's real product shape: local execution, durable worktrees, GitHub + Linear routing, and repair loops around PR truth.
